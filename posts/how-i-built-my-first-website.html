How I Built My First Website

January 10, 2026 6 min read

Everyone has a "first website" story. Mine was probably like yours - a single HTML file with some inline CSS, hosted somewhere free, that I showed to exactly three people (my mom, my best friend, and myself). Looking back, it's embarrassing, but it's also the foundation of everything I do now. Here's how it happened.

The Spark

I was in my first year of university when I realized that the courses weren't going to teach me everything I needed to know. The lectures on data structures were interesting, but I wanted to make things that people could actually use. Web development seemed like the perfect starting point - visible, accessible, and something I could learn on my own.

I don't remember exactly what inspired me, but I remember the moment clearly: I was watching a YouTube video about how websites work, and the presenter said something like "anyone can make a website - you just need a text editor and a browser." That seemed too simple to be true. I decided to find out.

Learning HTML: The Bare Metal

My first step was HTML. I had no idea what it stood for (HyperText Markup Language, as I later learned), but I knew it was the building block of the web. I found a tutorial on W3Schools and started typing.

<!DOCTYPE html>
<html>
<head>
    <title>My First Website</title>
</head>
<body>
    <h1>Hello World!</h1>
    <p>This is my website.</p>
</body>
</html>

Seeing "Hello World!" appear in a browser for the first time was magical. It was the same feeling I imagine early programmers had when they saw their first output. I had made something that existed on the internet (well, on my computer that could be on the internet).

I spent the next few days adding more content: headings, paragraphs, lists, links to other pages (that didn't exist yet). I learned about attributes, the difference between block and inline elements, and why semantic HTML matters (though I didn't appreciate that last part at the time).

Discovering CSS

My first website was ugly. Very ugly. Black text on white background, no spacing, no colors, no personality. It looked like a Word document from 1995. I knew there had to be a better way.

CSS was the answer. I learned about selectors, properties, and values. I discovered that I could change colors, fonts, and layouts. I made my first gradient background (cringe-worthy, in retrospect), added shadows to elements, and centered things using the magical "margin: 0 auto".

The most important lesson I learned about CSS was the box model. Understanding that every element is a box with content, padding, border, and margin unlocked the ability to create layouts that didn't look like they were designed by a bored teenager (even though they were).

The First Real Project

After a few weeks of learning, I decided to build something real. Not just a practice page, but something that had a purpose. I chose to create a simple fan page for my favorite game at the time.

This project taught me how to:

  • Plan a website structure before coding
  • Create navigation between pages
  • Use relative paths for internal links
  • Optimize images for web (poorly, but I tried)
  • Test across different browsers

The result was a five-page website with a navigation bar, some screenshots, character descriptions, and a contact form (that didn't actually work, but looked real). I was proud of it.

Version Control: The Game Changer

Around this time, I discovered Git. The concept of version control was new to me - tracking changes, creating branches, merging code. It seemed complicated, but I forced myself to learn it because every tutorial I watched mentioned it.

GitHub became my home for all my projects. It was (and still is) an incredible platform for sharing code, collaborating with others, and maintaining a portfolio. My early GitHub is full of abandoned projects and messy commit messages, but it's proof of my learning journey.

Hosting: Going Live

The final piece was hosting. I needed my website to be accessible to the world, not just my local machine. GitHub Pages was the answer - free hosting for static websites, directly connected to my GitHub repositories.

The process was simple:

  1. Create a repository on GitHub
  2. Push my HTML, CSS, and images
  3. Enable GitHub Pages in settings
  4. Wait a few minutes for deployment

When I first saw my website live at username.github.io, I shared the link with everyone I knew. It wasn't much, but it was mine, and it was on the real internet.

What I Would Do Differently

If I could go back and give myself advice, I would say:

Learn the fundamentals thoroughly. I rushed through HTML and CSS to get to the "fun" parts. This meant I had gaps in my knowledge that took years to fill. Understanding the DOM, accessibility, and responsive design early would have saved me time later.

Build projects, not tutorials. Tutorials are great for learning, but they have diminishing returns. After completing 5-10 tutorials, I should have started building real projects. Making mistakes in my own code taught me more than following someone else's code.

Share your work early. I was embarrassed by my early work and kept it hidden. Looking back, the quality didn't matter as much as the habit of sharing. Feedback, even negative, is valuable for growth.

Where It Led Me

That first website led to a second, then a third. Each one was better than the last. I learned JavaScript, then React. I started contributing to open-source projects. I built web applications for university assignments and personal projects.

Today, web development is a core part of my skill set. I build full-stack applications, create responsive designs, and understand the complexities of modern web architecture. It all started with a single HTML file and a desire to learn.

If you're starting your web development journey, my advice is simple: start now, make things, break things, and don't worry about perfection. Your first website won't be great, but it will be the foundation for something better.
