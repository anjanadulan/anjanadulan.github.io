My VS Code Setup: Extensions, Themes, and Productivity

January 15, 2026 7 min read By Anjana Wijerathna

VS Code is my second home. Here's my complete setup that took months to refine - extensions that actually help, a theme that's easy on the eyes, and settings that speed up my workflow.

Why I Chose VS Code

I tried Sublime Text first. It was fast and lightweight, but the plugin ecosystem felt scattered. Then came Atom, which was more modern but slower than I wanted. Finally, VS Code - Microsoft's free, open-source editor that strikes the perfect balance.

VS Code has excellent Git integration, a powerful extension marketplace, IntelliSense for smart code completion, and integrated terminal. It's customizable without being overwhelming, and it runs well even on my laptop.

The Theme: Dracula Official

I stare at my code editor for hours. The theme matters more than most people think. After trying dozens, I settled on Dracula Official.

Why Dracula? The syntax highlighting is distinct and beautiful. Comments are muted but readable. Strings are distinguishable from functions at a glance. The background is dark enough to reduce eye strain but not pure black, which feels harsh.

My settings.json tweaks for the theme:

{
  "editor.fontFamily": "'Fira Code', Consolas, 'Courier New', monospace",
  "editor.fontLigatures": true,
  "editor.fontSize": 14,
  "editor.lineHeight": 24,
  "editor.cursorBlinking": "smooth",
  "editor.cursorSmoothCaretAnimation": "on"
}

Fira Code with ligatures is beautiful. Ligatures combine common character combinations (like => or !==) into single, more readable glyphs.

Essential Extensions

Prettier - Code Formatter: The one extension everyone needs. Prettier enforces consistent formatting across your team. I set it to format on save, so my code is always clean without effort.

ESLint: Catches syntax errors and enforces best practices in JavaScript. It nags me about unused variables and missing semicolons, but that's why my code is clean.

GitLens: Supercharges the built-in Git integration. I can see who last modified a line, browse commit history visually, and compare branches without leaving the editor.

Live Server: For web development, this is gold. It launches a local development server with live reload. Every time I save an HTML or CSS file, the browser updates automatically.

Auto Rename Tag: When I rename an HTML opening tag, it automatically renames the closing tag. Such a simple thing, but it eliminates a common source of bugs.

Bracket Pair Colorizer 2: Makes matching brackets have the same color. In deeply nested code, this is a lifesaver for seeing where blocks start and end.

Path Intellisense: Autocompletes filenames when importing. Type import something from './ and it shows all files in that directory.

Error Lens: Shows errors and warnings inline rather than just in the problems panel. Errors are impossible to ignore.

Productivity Extensions

Turbo Console Log: Select a variable, press Ctrl+Alt+L, and it creates a console.log statement. Saves time when debugging.

Todo Tree: Finds all TODO and FIXME comments in your code and displays them in a tree view. Great for tracking things you need to come back to.

GitHub Copilot: AI-powered code completion that suggests entire lines or functions based on context. It's like having a pair programmer. Not perfect, but often helpful.

Bookmarks: Mark important lines and jump between them with keyboard shortcuts. Essential for navigating large files.

Language-Specific Extensions

Beyond the general extensions, I have language-specific tools:

  • Java Extension Pack: For Java development with debugging and IntelliSense
  • PHP IntelliSense: Autocomplete and refactoring for PHP
  • Python: Microsoft's official Python extension for data science and automation
  • Live Sass Compiler: Compiles Sass/SCSS to CSS in real-time

My Key Keyboard Shortcuts

Mouse is slow. Learning keyboard shortcuts changed my coding speed dramatically.

Ctrl + P           # Quick Open file
Ctrl + Shift + P   # Command Palette
Ctrl + D           # Select next occurrence of word
Ctrl + Shift + L   # Select all occurrences
Alt + Up/Down      # Move line up/down
Ctrl + /           # Toggle comment
Ctrl + B           # Toggle sidebar
Ctrl + J           # Toggle terminal
Ctrl + K Ctrl + S  # Keyboard Shortcuts

The most impactful shortcuts are Quick Open (Ctrl+P) and multi-cursor editing (Ctrl+D). I use them hundreds of times daily.

My settings.json

Here's the full configuration I use. You can access this via Command Palette > Preferences: Open Settings (JSON):

{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.minimap.enabled": false,
  "editor.wordWrap": "on",
  "files.autoSave": "afterDelay",
  "files.autoSaveDelay": 1000,
  "terminal.integrated.fontSize": 13,
  "workbench.colorTheme": "Dracula",
  "explorer.confirmDelete": false,
  "git.enableSmartCommit": true
}

The important ones: format on save (no手动 formatting), no minimap (personal preference, some love it), word wrap for long lines, auto save after 1 second delay.

Workspace Settings

Beyond user settings, I use workspace-specific settings for some projects. Create a .vscode folder in your project with settings.json to override global settings for that project.

This is useful when different projects have different formatting rules or need specific extensions enabled.

The Terminal Setup

VS Code's integrated terminal is powerful. I use PowerShell on Windows with Oh My Posh for a custom prompt that shows Git branch, Python virtual environment, and other useful info.

A good terminal setup matters because you'll spend significant time there - running builds, Git commands, npm scripts, and server processes.

Final Thoughts

Your editor setup should evolve. What works for me might not work for you. The key is experimenting: try an extension for a week, keep it if it helps, remove it if not.

Invest time in learning your tools. The 30 minutes you spend memorizing shortcuts will pay back hours over the months and years you'll use the editor.
