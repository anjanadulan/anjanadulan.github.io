Essential Tools for Every Developer: My Development Setup

January 25, 2026 6 min read By Anjana Wijerathna

After years of experimenting with different tools, I've settled on a development setup that works for me. Here's what's in my toolkit and why each piece matters.

The Foundation: Code Editor

Your code editor is where you'll spend most of your time, so choosing the right one matters. After trying Sublime Text, Atom, and even Vim for a while, I settled on Visual Studio Code as my primary editor.

What makes VS Code work for me:

Extensions: The extension ecosystem is incredible. Whether I need syntax highlighting for a new language, an integrated terminal, or a Git GUI, there's likely an extension for it. My must-have extensions include Prettier for formatting, ESLint for JavaScript linting, GitLens for version control, and Live Server for web development.

Customization: I spent an afternoon configuring my VS Code setup and haven't looked back. A cohesive dark theme (Dracula or One Dark Pro), custom font with ligatures (Fira Code), and carefully chosen settings make coding more pleasant.

Integrated Terminal: Having a terminal right in the editor saves context-switching. I split panes between my editor and terminal, running build commands and Git operations without leaving the window.

Version Control: Git and GitHub

Version control isn't optional anymore - it's essential. Git tracks changes to your code, allows collaboration, and provides a safety net when things go wrong.

I use Git through the command line primarily, but having a GUI tool helps visualize branches and history. GitHub Desktop is simple and effective for most operations, while SourceTree offers more advanced features when needed.

Key Git commands I use daily:

git add .                 # Stage all changes
git commit -m "message"  # Commit with a message
git push                 # Push to remote
git pull                 # Pull latest changes
git status               # Check current state
git log --oneline       # View commit history
git branch               # List branches
git checkout -b new      # Create and switch to new branch

The workflow I've adopted: create a branch for each feature, commit often with meaningful messages, and merge via pull requests after code review. This keeps the main branch clean and provides a history of decisions.

Database Tools

Working with databases is part of most projects. For MySQL, I use MySQL Workbench for designing schemas and running queries. The visual query builder makes complex joins less intimidating, and the ERD diagrams help plan database structure.

For MongoDB, MongoDB Compass provides a GUI that makes browsing collections and running aggregation pipelines intuitive. Seeing data visually rather than as JSON documents helps me understand relationships.

When debugging, I'll connect directly to databases using these tools rather than relying solely on application queries. It helps verify data state and test queries before implementing them.

API Development and Testing

Building REST APIs requires testing them. Postman has become my go-to tool for API development. I can organize requests into collections, set environment variables, and automate test suites.

My typical workflow:

  • Design the API endpoints
  • Create a Postman collection for the API
  • Test each endpoint individually
  • Add tests to verify responses
  • Export the collection for documentation

This systematic approach catches bugs early and provides living documentation of how the API should work.

Browser DevTools

For web development, browser developer tools are indispensable. Chrome DevTools has become my primary debugging environment. The Elements panel helps debug CSS, the Console captures JavaScript output, and the Network tab shows request/response cycles.

Pro tip: Learn the shortcuts. Opening DevTools with F12, the Command Menu with Ctrl+Shift+P, and quick source file navigation with Ctrl+P save countless clicks throughout the day.

Productivity Tools

Beyond coding tools, these applications support my workflow:

Notion: For notes, project planning, and documentation. I have databases for tracking project ideas, keeping study notes, and organizing research.

Discord: Developer communities are invaluable. I hang out in programming servers where I can ask questions, share progress, and learn from others.

Rectangle (Mac) / Windows Snap: Window management tools. I always have VS Code, terminal, and browser visible simultaneously, and these tools make arranging windows effortless.

Spotify: Lo-fi music helps me focus during deep work sessions. Having background music without lyrics reduces distractions while keeping the atmosphere pleasant.

Documentation and References

Every developer needs quick access to documentation. I've bookmarked:

  • MDN Web Docs for web technologies
  • JavaDocs for Java
  • Node.js documentation
  • MongoDB manual
  • GitHub documentation

Rather than memorizing everything, I memorize where to find information. The documentation is always a search away.

Final Thoughts

Your tools should serve you, not the other way around. Don't feel obligated to use what others use - experiment, find what clicks for you, and stick with it. My setup has evolved over years, and it'll continue changing as I encounter new challenges.

The best tool is the one that gets out of your way and lets you build. Everything else is just preference.
